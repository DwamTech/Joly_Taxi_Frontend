# صفحة إدارة البلاغات

## الوصف
صفحة إدارة البلاغات تتيح للمسؤولين التعامل مع بلاغات المستخدمين ومتابعة حالتها.

## المكونات

### 1. ReportsHero
- عرض إحصائيات البلاغات:
  - إجمالي البلاغات
  - البلاغات المعلقة
  - البلاغات المحلولة
  - البلاغات اليوم
- خلفية داكنة مثل الصفحة الرئيسية

### 2. ReportsFilters
- البحث برقم البلاغ، اسم المُبلِّغ، اسم المُبلَّغ عنه
- فلترة حسب:
  - الحالة (معلق، محلول)
  - السبب
  - الأولوية (عالية، متوسطة، منخفضة)
  - تاريخ البلاغ (من - إلى)
- الترتيب (الأحدث، الأقدم، الأولوية)

### 3. ReportsTable
- عرض قائمة البلاغات مع:
  - رقم البلاغ (ID)
  - الرحلة (رقم + رابط)
  - المُبلِّغ (اسم + نوع)
  - المُبلَّغ عنه (اسم + نوع)
  - السبب
  - الحالة (Pending/Resolved)
  - الأولوية (عالية/متوسطة/منخفضة)
  - تاريخ البلاغ
  - الإجراءات (عرض، حل، حذف)
- هيدر الجدول بنفس تصميم جدول المستخدمين

### 4. ReportDetailsModal
- عرض تفاصيل البلاغ الكاملة:
  - رقم البلاغ والحالة والأولوية
  - تاريخ البلاغ وتاريخ الحل
  - معلومات الرحلة (رقم + رابط + تفاصيل)
  - معلومات المُبلِّغ:
    - الاسم (مع زر عرض تفاصيل المستخدم)
    - النوع (راكب/سائق)
    - رقم الهاتف
    - عدد البلاغات السابقة
  - معلومات المُبلَّغ عنه:
    - الاسم (مع زر عرض تفاصيل المستخدم)
    - النوع (راكب/سائق)
    - رقم الهاتف
    - عدد البلاغات المستلمة
    - التقييم العام
  - السبب والوصف التفصيلي
  - ملاحظات الإدارة (قابلة للتعديل)
  - الإجراء المتخذ (قابل للتعديل)
- الإجراءات المتاحة:
  - حفظ الملاحظات
  - وضع علامة كمحلول
  - حظر المُبلَّغ عنه
  - إرسال تحذير
  - إرسال إشعار للطرفين

## البيانات

### Model: TripReport
```typescript
{
  id: number;
  trip_request_id: number;
  reporter_id: number;
  reported_id: number;
  reason: string;
  description?: string;
  status: "pending" | "resolved";
  priority: "high" | "medium" | "low";
  admin_notes?: string;
  action_taken?: string;
  resolved_at?: string;
  created_at: string;
  updated_at: string;
}
```

- خلفية: `linear-gradient(135deg, #1a1a1a 0%, #2d2d2d 100%)`

- مثل جدول المستخدمين

- متوسطة: `#f57c00` (برتقالي)
